# Codex 発注書：薄明GreenField サイト外殻 v0 実装

## 目的

個人サイト **薄明GreenField** の公開用フロントエンドの「ガワ」を実装してください。

すでにホーム画面のモックアップとして、リポジトリ内に `mockUp.html` が存在します。
今回の作業では、この `mockUp.html` をデザイン・構成の基準として、静的サイトとして運用可能な形に整理・実装してください。

最終的には Cloudflare Pages でホスティングする想定です。

---

## 前提

* 既存の `mockUp.html` を必ず確認すること。
* `mockUp.html` の見た目・雰囲気・画面構成を尊重すること。
* 完全なピクセル一致は不要だが、構成・情報設計・世界観は維持すること。
* サイトは静的サイトとして成立させること。
* バックエンド機能は今回実装しない。
* 記事投稿機能、ログイン機能、AI自動記事生成機能、ハコニワ本体は今回の対象外。
* ただし、それらに将来つながる導線・プレースホルダーは用意する。

---

## 技術方針

既存プロジェクトの構成を確認したうえで、以下の方針で進めてください。

### 既存の Astro / 静的サイト構成がある場合

* 既存構成を優先して利用する。
* `mockUp.html` の内容を Astro コンポーネント化する。
* ページ・レイアウト・共通部品に分割する。

### まだ静的サイト構成がない場合

* Astro ベースの静的サイトとして最小構成を作成する。
* Cloudflare Pages でのデプロイを想定した構成にする。
* 不要に複雑なフレームワークやバックエンド依存を入れない。

---

## 実装対象

### 1. ホーム画面

`mockUp.html` をもとに、トップページを実装してください。

想定構成は以下です。

* Header

  * ロゴ / サイト名
  * 主要リンク
  * モバイル用メニュー
  * 設定ボタンまたは設定アイコンのプレースホルダー

* Ticker / お知らせ欄

  * サイト更新情報や短い告知を表示する領域
  * 現時点ではダミーテキストでよい

* Hero / メイン導入エリア

  * 薄明GreenField の概要
  * 個人ブログ・作品置き場・実験場としての説明
  * 「記事」「ツール」「ゲーム」などへの導線

* Section Grid

  * Human Article
  * System Article
  * Tools
  * Games
  * Novel
  * ハコニワ
  * それぞれカード形式で表示
  * 未実装のものは `coming soon` 表記でよい

* Footer

  * サイト名
  * コピーライト
  * 主要リンク
  * 簡単な説明文

---

## 2. ルーティング / プレースホルダーページ

ホーム画面から遷移できる最低限のページを作ってください。

以下のルートを用意してください。

```txt
/
 /articles
 /articles/h
 /articles/s
 /tools
 /games
 /novel
 /hakoniwa
 /about
```

各ページは、現時点では簡単なプレースホルダーで構いません。

ただし、以下のように将来の役割が分かる説明を入れてください。

### `/articles`

記事全体の入口。

* Human Article
* System Article

への導線を配置。

### `/articles/h`

人間が書いた記事の一覧予定地。

### `/articles/s`

2A / s-article による自動生成・半自動生成記事の一覧予定地。

現時点では未実装でよい。

### `/tools`

変換器・便利ツール類の置き場予定地。

### `/games`

ブラウザゲーム・小作品の置き場予定地。

### `/novel`

ノベル・創作コンテンツ置き場予定地。

現時点では `coming soon` でよい。

### `/hakoniwa`

ハコニワ計画の入口。

以下の導線を将来的に置ける構成にしてください。

* ハコニワ概要
* ハコニワ管理記
* ハコニワ日記

今回はプレースホルダーで構いません。

### `/about`

薄明GreenField の概要ページ。

## 3. コンポーネント分割

可能であれば、以下のようにコンポーネント化してください。

```txt
src/
  components/
    Header.astro
    Footer.astro
    Hero.astro
    Ticker.astro
    SectionCard.astro
    SectionGrid.astro
  layouts/
    BaseLayout.astro
  pages/
    index.astro
    articles/
      index.astro
      h.astro
      s.astro
    tools.astro
    games.astro
    novel.astro
    hakoniwa.astro
    about.astro
```

既存構成がある場合は、無理にこの構造へ完全一致させなくてよいです。
ただし、今後の拡張がしやすい構成にしてください。

---

## 4. デザイン要件

`mockUp.html` のデザインを基準としてください。

特に以下を重視してください。

* 薄明・夕暮れ・緑場・個人サイトらしさがあること
* 古すぎないが、過度に企業サイト風にしないこと
* 個人ブログ、作品置き場、実験場が同居する雰囲気にすること
* PC / タブレット / スマホで破綻しないこと
* モバイルではカードやメニューが自然に縦積みになること
* 画像やサムネイルを後から追加しやすいこと
* テキスト差し替えが容易なこと

---

## 5. レスポンシブ対応

以下の表示を最低限確認してください。

* PC 幅
* タブレット幅
* スマホ幅

スマホ表示では、以下を満たしてください。

* 横スクロールが発生しない
* Header が破綻しない
* Section Grid が1列または見やすい列数になる
* 文字が小さすぎない
* タップ対象が狭すぎない

---

## 6. 実装しないこと

以下は今回の作業範囲外です。

* 記事エディタ
* 記事投稿機能
* CMS
* ログイン機能
* Discord連携
* 2A自動記事生成
* ハコニワ実行環境
* 検索機能
* コメント機能
* 多言語翻訳の本実装
* 江戸スタイル / 平安スタイルの本実装

ただし、設定ボタンや将来の切り替えUIを置くための空きは残して構いません。

---

## 7. テキストについて

文言は仮で構いませんが、以下の雰囲気にしてください。

* サイト名：薄明GreenField
* 個人ブログ / 作品置き場 / 実験場
* 人間が書く記事と、システムが支援する記事の二系統がある
* ツール・ゲーム・ノベル・ハコニワなどを置く
* まだ準備中の領域があることを自然に示す

例：

```txt
薄明GreenFieldは、佐用瑠璃が運営する個人ブログ兼実験場です。
記事、ツール、ゲーム、ノベル、そしてAI実験場としてのハコニワを少しずつ整備していきます。
```

## 8. 成果物

作業完了時に、以下を満たしてください。

* `mockUp.html` をもとにしたホーム画面が実装されている
* 各主要ページへのリンクが機能している
* プレースホルダーページが存在する
* PC / スマホで表示が破綻しない
* ビルドが通る
* 不要なサンプルコードが残っていない
* README かコメントで、起動方法とビルド方法が分かる

## 9. 確認コマンド

Astro 構成の場合、以下が通るようにしてください。

```bash
npm install
npm run dev
npm run build
```

既存プロジェクトのコマンドが異なる場合は、既存の `package.json` に合わせてください。

以下を満たしたら完了とします。

* トップページが `mockUp.html` の意図を反映している
* 主要導線がすべて存在する
* 未実装領域は `coming soon` として自然に処理されている
* Cloudflare Pages に載せる前段階として、静的ビルドが成功する
* コードが今後の記事・ツール・ゲーム追加に耐える程度に整理されている
